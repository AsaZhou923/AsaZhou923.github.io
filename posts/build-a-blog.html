从 0 搭一个个人博客：我希望它首先是一个“能长期写下去”的地方

  • 2026.06.28
  • 博客
  • 静态站点
  • 数字花园

我不想把个人博客做成一个需要维护的“作品集”。 我更希望它像一个工作台：打开就能写，写完就能发布， 过几个月再回来仍然能读懂。所以我先设计的不是视觉，而是结构。

一个能长期写下去的博客，首先要降低开始写的阻力。

为什么先做结构

视觉会影响第一印象，但结构决定我能不能持续使用它。 如果写一篇文章需要想清楚分类、路径、模板、摘要和发布方式， 那么这个系统就会慢慢变成负担。

所以我把第一版控制在几个稳定页面：首页、文章、项目、归档和关于。 首页负责说明“这里是什么”，文章负责完整表达，项目负责展示实践结果， 归档负责让旧内容还能被重新找到。

博客和数字花园

博客更像公开发表，数字花园更像持续整理。前者强调完成，后者允许生长。 我希望这个站点同时容纳两种状态：有些内容可以是完整文章， 有些内容只是当下的一段观察。

  1. 文章：问题清楚、论证完整、可以被外部读者直接理解。
  2. 笔记：还在生长的想法，用来保存过程和链接。
  3. 项目：把想法落地后的结果，也记录过程里的取舍。

最小可用页面

第一版不需要复杂 CMS，也不需要评论系统。它只需要能被搜索引擎读取， 能被 RSS 订阅，能在 GitHub Pages 上稳定访问，并且在手机上阅读舒服。

对我来说，个人博客不是一次性展示，而是一套写作习惯的容器。 只要这个容器足够轻，它就有机会被长期使用。
